Making the "Keeping Track" app

In this "blog" or whatever this is. I will write down my challenges and how I solved them. While trying to make an app for android where I can fill in statistics of each day, such as waking up time and sleeping, and what I ate that day. A journal sort of.

Some info

So for this project I will use Flutter. A framework by google for making crosscompatible apps.
And for saving the users data I will use firebase auth with firestore.

Learning basics

Note that I've never used Flutter or dart before, So before I could dive in blindly I thought it would be smart to get to know the basics.

Making the actual app

Yes, I know the previous paragraph was short, I got impatient and wanted to start on the actual app right away.

Creating the structure

So for this app I just simply want 4 pages to start with.

  • Home/Days
  • Day info
  • Edit day
  • Login
  • Loading
I will explain more about these pages one by one.

Home/Days

So this is without a doubt the most important page, This page will contain a list of every day reported on small cards that give a summary. It will also have a button in the bottom right corner. I have made a sketch in paint.net to get an idea of how I want it to look:


Day info

So this will be the page that pops up when you click on a Day card in the Home/Days page, This will contain full information that has been filled in by the user. I also made a sketch for this page:


Edit day

This page will be almost exactly the same as the Day info page, But it will be possible to edit the values.


Login

Now the login page, This page only has 1 purpose, which is making sure the user logins so firestore database can save their data.


Loading

I don't think I need to explain much here, This page is just a simple page when anything needs to load. And it has a fun little circle load thingy


Creating the authentication / login

So in this part I will try to make the user successfully login with google, For this I use firebase auth. I mostly just followed This great tutorial, I did struggle quite a lot with getting that to work ( ugh ugh took 4 hours), But this stackoverflow thread has helped me a ton with resolving those issues.
But after I had this working, I honestly still had no idea how it worked, and how I could use it to store a users data.
So at this point I made a quick flowchart to better understand what has to happened and when:

So I successfully did the Login Page and had gotten the User ID,
Now I had to send this data to the Loading page.

So I did that, It now sends the user ID to the loading page, and from the loading page it gets the users Data from the database.
And from there on, It sends that userData to the home/days page.
